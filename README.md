## CS 255 Portfolio Reflection – DriverPass Project

The DriverPass project was based on a client who wanted a system to help students prepare for their driving tests. The system needed to allow users to register, schedule driving lessons, take practice tests, and track their progress. It also included features for employees, such as managing appointments and generating reports. Overall, the goal was to design a system that improves how students learn and prepares them more effectively for the DMV exam.

This project was based on the DriverPass interview and requirements provided here:
https://learn.snhu.edu/d2l/lor/viewer/viewFile.d2lfile/2200206/24276,-1/

One thing I did particularly well was organizing the requirements and translating them into clear system components. I made sure the diagrams and documents connected logically, especially the use case diagram and system design, so everything aligned with what the client needed.

If I could revise one part of my work, I would improve the level of detail in my diagrams, especially the activity and sequence diagrams. I would make them more precise and include clearer labels so that someone unfamiliar with the project could easily understand the flow without extra explanation.

To interpret the user’s needs, I focused on the interview transcript and identified the main features the client requested. I then translated those into system requirements and design elements. Considering the user’s needs is important because the system is being built for them. If their needs are misunderstood, the final product may not be useful or effective.

When designing software, I usually start by understanding the problem and gathering requirements. Then I break the system into smaller parts using diagrams like use case and class diagrams. In the future, I plan to spend more time refining requirements early and using iterative design so I can improve the system step by step.
